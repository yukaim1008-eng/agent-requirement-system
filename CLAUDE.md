# 多 Agent 协作 —— 自动化需求分析系统

## 一句话定位
5 个 AI Agent 分工协作，把一句模糊需求变成专业的需求规格说明书（PRD）。

## 5 个 Agent 角色
| 角色 | 干嘛的 | 产出 |
|------|--------|------|
| 协调员 | 拆任务→调度→整合 | 调度日志 |
| 需求分析师 | 挖需求→生成用户故事 | 用户故事列表 |
| 行业研究员 | 调 Tavily 搜竞品+法规 | 竞品摘要 |
| 技术评估员 | 评可行性+风险+工期 | 评估报告 |
| 文档专员 | 汇总→导出 Markdown/Word | PRD 文档 |

## 高阶玩法
- Agent 互质疑：技术评估发现不可行→退回需求分析师改
- 人在回路：关键节点人工确认
- 真实搜索：Tavily API，不是 LLM 模拟
- 联动：技术评估员调 it-ops-rag 的高阶 RAG
