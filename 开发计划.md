# 多 Agent 需求分析系统 · 开发计划

> 一个**可编排、可干预、可复用**的 AI 工作流：把一句模糊需求，经 5 个分工 Agent 协作，产出结构化 PRD。
> 亮点：**Agent 互质疑 + 人在回路 + 跨系统联动（调用自建 RAG）**。

**目标**：一句模糊需求 → 5 Agent 协作 → 结构化 PRD（可下载 Word/Markdown）
**技术栈**：Python 3.13 · CrewAI · DeepSeek API · Tavily · Streamlit · python-docx

---

## 三个工程决策

| 点 | 方案 | 为什么 |
|----|------|--------|
| LLM Key | 1 个 DeepSeek key 驱动全部 5 Agent，靠 System Prompt 区分角色 | 零额外成本（多 Agent ≠ 多 Key） |
| RAG 联动 | 子进程拉起 RAG 的 `backend.py`，走 JSON 行协议通信 | venv/重模型完全隔离，复用现成接口 |
| Tavily | 有 key 走真实搜索，无 key 自动 LLM 模拟兜底 | 先跑通，注册后无缝切换 |

## 文件结构

| 文件 | 职责 |
|------|------|
| `config.py` | 5 个角色 System Prompt、`get_llm()`、Tavily/RAG 开关 |
| `agents.py` | 5 个 Agent 定义 + 工具绑定 + `allow_delegation` |
| `tasks.py` | 各 Task + `context` 传递 + `human_input` 干预点 |
| `tools.py` | Tavily 搜索（降级）、PRD 导出、RAG 查询（子进程） |
| `crew_runner.py` | 编排入口、流程控制、互问环 |
| `utils.py` | 日志、状态、输出格式化 |
| `main.py` | Streamlit：对话气泡、进度、HITL 确认、PRD 下载 |

## 分阶段

- **阶段 0 · 环境**：venv + 依赖 + `.env`/`.gitignore`。验证 `import crewai`
- **阶段 1 · 配置 + Agent**：5 角色 Prompt 集中 config，`get_llm()`，实例化 5 Agent
- **阶段 2 · 串行跑通**：`Process.sequential` 需求→研究→评估→文档，`context` 传递。验证产出完整 PRD 文本
- **阶段 3 · 工具集成**：RAG 子进程查询（含降级）、Tavily（降级）、PRD 导出 docx
- **阶段 4 · 高阶**：`Process.hierarchical` 实现 Agent 互质疑；`human_input=True` 实现人在回路
- **阶段 5 · 界面+打磨**：Streamlit 气泡/进度/确认/下载、错误降级、README、Demo

## 三个差异化亮点（面试对着讲）

1. **Agent 互质疑**：技术评估员发现需求不可行 → 退回需求分析师修改（hierarchical + delegation）
2. **人在回路**：关键节点（需求优先级）暂停等人工确认（human_input）
3. **跨系统联动**：技术评估员子进程调用自建 RAG 查历史经验（进程隔离 + JSON 协议）
